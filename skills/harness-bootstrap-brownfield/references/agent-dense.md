# Agent-dense writing (brownfield map)

`brownfield-map.md` is **cold context for agents**, not a human essay.

## Rules

1. **Tables and one-liners only** — no overview paragraphs, no narrative recommendations.
2. **No stack duplication** — when `architecture-rules.md` exists, link it; add a stack row only for deltas or when the constitution is missing/stub.
3. **Counts over prose** — module rows use numbers (or `—`); put caveats in a single `Notes` cell.
4. **Gaps as a priority table** — P1/P2/P3; drop "Recommendations for next planning" essays (keep ≤3 bullets under Next planning).
5. **Target size** — prefer ≤120 lines; hard soft-cap ~180.
