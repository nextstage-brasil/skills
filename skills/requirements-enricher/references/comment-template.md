# Comment template (GitLab internal note)

Copy structure below into `add_issue_comment` `body`. Replace `{placeholders}`.

**Verdict icon (mandatory):** first character of the first line — `❌` if there are blocking questions; `✅` if zero blocking questions (execution-ready).

**Questions:** always for the **issue author (requester)** — plain product language. No schema, branch, labels, class names, or file paths inside the numbered list.

**`{author_username}`:** copy **only** `author.username` from `read_issue` (GitLab login). Never `author.name`, never slugify the display name, never guess.

## Variant A — blocking questions (`❌`)

```markdown
❌ @{author_username} — requirements enrichment for autonomous execution. **{N} blocking question(s).**

## Current understanding

{2–5 bullets: goal in product terms; optional short note of constraints}

**Assumptions adopted (confirm if incorrect):**
- {product or technical assumption the agent will use if unanswered}
- {…}

**Areas investigated in code:** `{path1}`, `{path2}`, …

## Questions

{numbered list — requester-facing only}

1. {plain-language question, e.g. when does the filter apply?}
2. {…}

---

_Requirements enrichment generated by requirements-enricher. Reply in this issue (you may number answers: `1. …`, `2. …`)._
```

## Variant B — execution-ready (`✅`)

```markdown
✅ @{author_username} — requirements enrichment for autonomous execution. **Ready for execution.**

## Current understanding

{summary}

## Status

Requirements are **sufficient** for autonomous execution with the assumptions above. No blocking questions pending.

**Suggested next step:** `execute-gitlab-issue` with this issue.

---

_Requirements enrichment generated by requirements-enricher._
```
